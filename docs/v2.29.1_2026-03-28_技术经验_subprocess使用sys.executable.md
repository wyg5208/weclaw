# 技术经验：subprocess 调用 Python 子进程应使用 sys.executable

**版本**: v2.29.1  
**日期**: 2026-03-28  
**类型**: 技术经验记录  
**问题标签**: subprocess、虚拟环境、Python 解释器路径

---

## 1. 问题背景

在 WeClaw 项目中，PDF 生成工具需要使用 Playwright 进行 HTML 转 PDF。某些场景下需要通过 `subprocess` 调用 Python 子进程来执行特定任务。

### 1.1 问题代码位置

**文件**: `src/tools/pdf_generator.py` 第 549 行

```python
# 原始代码（错误写法）
subprocess.run(["python", "-c", ...], ...)
```

### 1.2 问题代码完整上下文

```python
# src/tools/pdf_generator.py

def some_pdf_generation_function(...):
    # ... 其他代码 ...
    
    # 第 549 行附近的代码
    result = subprocess.run(
        ["python", "-c", code_content],
        capture_output=True,
        text=True,
        timeout=timeout
    )
```

---

## 2. 问题现象

### 2.1 症状描述

在虚拟环境（venv）中运行 WeClaw 时，PDF 生成功能在某些场景下失败。错误日志显示：

```
ModuleNotFoundError: No module named 'playwright'
```

或

```
playwright 模块无法找到，尽管已在虚拟环境中安装
```

### 2.2 环境差异

| 运行环境 | 主进程 Python | subprocess "python" 指向 |
|---------|--------------|--------------------------|
| 系统全局 Python | `C:\Python311\python.exe` | 系统 PATH 中的 `python` |
| 虚拟环境 venv | `D:\projects\weclaw\venv\Scripts\python.exe` | 系统 PATH 中的 `python`（非 venv） |

**关键问题**：主进程使用虚拟环境中的 Python，但 subprocess 使用系统 PATH 中的 `python`，两者不一致！

---

## 3. 根本原因分析

### 3.1 subprocess 查找机制

当使用 `subprocess.run(["python", ...])` 时：

1. `subprocess` 从系统环境变量 `PATH` 中查找名为 `python` 的可执行文件
2. 系统 PATH 通常指向全局 Python 安装目录
3. **虚拟环境中的 Python 不在系统 PATH 中**

### 3.2 环境隔离机制

```
虚拟环境 (venv) 的隔离原理：
┌─────────────────────────────────────────┐
│  主进程 (WeClaw)                        │
│  Python: D:\venv\Scripts\python.exe     │
│  sys.path: 包含 venv 的 site-packages   │
│  可用: playwright, pypdf 等             │
└─────────────────────────────────────────┘

系统 PATH:
  C:\Python311\Scripts\
  C:\Python311\              ← 全局 Python
  D:\venv\Scripts\           ← venv 不在 PATH 中！
  
subprocess.run(["python", ...]):
  从 PATH 查找 → 找到 C:\Python311\python.exe
  → 使用全局 Python → 无 playwright 模块
```

### 3.3 风险评估

| 场景 | 主进程 Python | subprocess Python | 是否正常 |
|------|--------------|-------------------|----------|
| 无虚拟环境 | 系统 Python | 系统 Python | ✅ |
| 激活虚拟环境 | venv Python | 系统 Python | ❌ |
| PyInstaller 打包 | 内置 Python | 系统 Python | ❌ |
| Docker 容器 | 容器 Python | 宿主机 Python | ❌ |

---

## 4. 解决方案

### 4.1 核心原则

> **始终使用 `sys.executable` 而不是硬编码的 "python" 命令**

### 4.2 修复代码

**修改 1**: 在文件顶部添加 `import sys`

```python
# src/tools/pdf_generator.py

import sys
import subprocess
# ... 其他 imports
```

**修改 2**: 将第 549 行的代码改为使用 `sys.executable`

```python
# 修改前（错误）
result = subprocess.run(
    ["python", "-c", code_content],
    capture_output=True,
    text=True,
    timeout=timeout
)

# 修改后（正确）
result = subprocess.run(
    [sys.executable, "-c", code_content],
    capture_output=True,
    text=True,
    timeout=timeout
)
```

### 4.3 验证修复

```python
import sys

print(f"主进程 Python: {sys.executable}")
# 输出: D:\python_projects\weclaw\venv\Scripts\python.exe

result = subprocess.run(
    [sys.executable, "-c", "import sys; print(sys.executable)"],
    capture_output=True,
    text=True
)
print(result.stdout.strip())
# 输出: D:\python_projects\weclaw\venv\Scripts\python.exe
# ✅ 子进程与主进程使用同一个 Python 解释器
```

## 5. 最佳实践总结

### 5.1 subprocess 调用 Python 的规范写法

```python
import sys
import subprocess

# ✅ 推荐写法
subprocess.run([sys.executable, "-c", "some_code"])
subprocess.run([sys.executable, "-m", "module_name"])

# ❌ 避免写法
subprocess.run(["python", "-c", "some_code"])
subprocess.run(["python3", "-c", "some_code"])
subprocess.run(["py", "-c", "some_code"])
```

### 5.2 完整示例

```python
import sys
import subprocess

def run_python_code(code: str, timeout: int = 30) -> subprocess.CompletedProcess:
    """
    在与主进程相同的 Python 环境中执行代码。
    
    Args:
        code: 要执行的 Python 代码字符串
        timeout: 超时时间（秒）
        
    Returns:
        CompletedProcess 对象
    """
    return subprocess.run(
        [sys.executable, "-c", code],
        capture_output=True,
        text=True,
        timeout=timeout,
        cwd=sys.path[0]  # 可选：设置工作目录
    )

def run_python_module(module: str, *args) -> subprocess.CompletedProcess:
    """运行 Python 模块"""
    cmd = [sys.executable, "-m", module] + list(args)
    return subprocess.run(cmd, capture_output=True, text=True)
```

### 5.3 特殊情况处理

#### 5.3.1 使用 shell=True 时

```python
# ❌ 危险写法
subprocess.run(f"python -c '{code}'", shell=True)

# ✅ 正确写法（不使用 shell=True）
subprocess.run([sys.executable, "-c", code])

# ✅ 如果必须使用 shell=True（不推荐）
subprocess.run(f'"{sys.executable}" -c "{code}"', shell=True)
```

#### 5.3.2 Windows 上使用 py 启动器

```python
# py 启动器会查找 shebang 或最新版本 Python
# 但无法保证使用虚拟环境中的 Python

# ❌ 不推荐
subprocess.run(["py", "-c", code])

# ✅ 推荐
subprocess.run([sys.executable, "-c", code])
```

#### 5.3.3 跨平台路径处理

```python
import sys
import shutil

# 确保 Python 解释器路径正确
python_path = sys.executable

# Windows: "D:\venv\Scripts\python.exe"
# Linux/macOS: "/home/user/venv/bin/python"

# subprocess 会自动处理路径格式
result = subprocess.run([python_path, "-c", "print('hello')"])
```

## 6. 相关技术点

### 6.1 sys.executable 详解

| 属性 | 说明 |
|------|------|
| `sys.executable` | 当前 Python 解释器的绝对路径 |
| `sys.version` | Python 版本信息 |
| `sys.prefix` | Python 安装前缀（虚拟环境根目录） |
| `sys.base_prefix` | 虚拟环境激活前的原始前缀 |

```python
import sys

# 虚拟环境中
print(sys.executable)     # D:\venv\Scripts\python.exe
print(sys.prefix)         # D:\venv
print(sys.base_prefix)    # C:\Python311 (全局 Python)

# 全局 Python 中
print(sys.executable)     # C:\Python311\python.exe
print(sys.prefix)         # C:\Python311
print(sys.base_prefix)    # C:\Python311
```

### 6.2 shutil.which() 备选方案

```python
import shutil

# 查找 python 在 PATH 中的位置
python_path = shutil.which("python")
print(python_path)  # C:\Python311\python.exe

# ⚠️ 但这仍然无法找到虚拟环境中的 Python
# 因为虚拟环境不在 PATH 中
```

### 6.3 环境变量继承

```python
import sys
import subprocess
import os

# subprocess 默认继承当前进程的环境变量
# 但可以显式指定

result = subprocess.run(
    [sys.executable, "-c", "import sys; print(sys.path)"],
    env={**os.environ, "PYTHONPATH": "/custom/path"}
)
```

## 9. 总结

### 9.1 核心经验

> **subprocess 调用 Python 子进程时，必须使用 `sys.executable` 而不是硬编码的 "python" 字符串，以确保子进程使用与主进程相同的 Python 解释器。**

### 9.2 一句话规则

```
subprocess.run(["python", ...]) → subprocess.run([sys.executable, ...])
```

### 9.3 适用场景

- ✅ 需要调用 Python 子进程的任何场景
- ✅ 虚拟环境 (venv) 中运行
- ✅ PyInstaller 打包的应用
- ✅ Docker 容器中运行
- ✅ 需要确保依赖包可用的场景

### 9.4 检查清单

在提交代码前，检查是否包含 subprocess 调用 Python 的代码：

- [ ] 是否使用了 `[sys.executable, ...]` 而非 `["python", ...]`
- [ ] 文件顶部是否导入了 `sys` 模块
- [ ] 是否在虚拟环境中测试过
