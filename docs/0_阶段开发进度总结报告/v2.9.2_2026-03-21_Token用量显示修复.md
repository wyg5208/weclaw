## 任务目标

修复桌面窗体右侧 Token 用量面板中，输入和输出 Token 数量始终显示为 0 的问题。费用显示正常，但输入/输出统计缺失。

## 问题诊断与分析

### 问题现象
- **现象**: 桌面端右侧面板 Token 用量显示异常
  - 输入 Token：始终显示 0
  - 输出 Token：始终显示 0
  - 费用：正常显示累计金额

### 根因分析

通过代码追踪发现：

1. **信号发射点硬编码为 0**
   - 文件：`src/ui/gui_app.py` 第 169 行
   - 问题代码：
     ```python
     self.usage_updated.emit(0, 0, cost)  # 输入、输出硬编码为 0
     ```
   - `usage_updated` 信号定义为 `Signal(int, int, float)`，分别对应 `input_tokens, output_tokens, cost`

2. **ModelRegistry 缺少聚合属性**
   - 文件：`src/models/registry.py`
   - `ModelRegistry` 类有 `total_cost` 和 `total_tokens` 属性
   - 但缺少 `total_prompt_tokens` 和 `total_completion_tokens` 的聚合统计

## 解决方案与实施

### 1. ModelRegistry 新增聚合属性

**文件**: `src/models/registry.py`

新增两个 property 方法，从历史调用记录中汇总统计：

```python
@property
def total_prompt_tokens(self) -> int:
    """所有历史调用的输入 Token 累计。"""
    return sum(r.prompt_tokens for r in self._usage_history)

@property
def total_completion_tokens(self) -> int:
    """所有历史调用的输出 Token 累计。"""
    return sum(r.completion_tokens for r in self._usage_history)
```

### 2. GuiAgent 修复信号发射

**文件**: `src/ui/gui_app.py`

修改 `chat` 方法中的用量更新逻辑：

```python
# 修复前
self.usage_updated.emit(0, 0, cost)

# 修复后
prompt_tokens = self._model_registry.total_prompt_tokens
completion_tokens = self._model_registry.total_completion_tokens
self.usage_updated.emit(prompt_tokens, completion_tokens, cost)
```

## 技术实现细节

### 数据流向

```
Agent.chat() 
  → ModelUsageEvent(prompt_tokens, completion_tokens, cost)
    → ModelRegistry._usage_history.append(UsageRecord)
      → ModelRegistry.total_prompt_tokens / total_completion_tokens
        → GuiAgent.usage_updated.emit()
          → MainWindow.update_usage()
            → UI 显示更新
```

## 经验教训

### 1. 硬编码陷阱
在信号发射等"看似简单"的地方，容易出现硬编码占位符（如 `0, 0`），导致功能失效。应始终使用真实数据源。

### 2. 数据聚合属性完整性
当数据类有多个字段时，聚合属性应覆盖所有常用字段。`ModelRegistry` 之前只有 `total_cost` 和 `total_tokens`，遗漏了更细粒度的 `prompt_tokens` 和 `completion_tokens`。

### 3. 调试思路
- 从 UI 显示层逆向追踪信号源
- 检查信号发射点的参数是否为真实数据
- 确认数据模型是否提供必要的聚合接口
