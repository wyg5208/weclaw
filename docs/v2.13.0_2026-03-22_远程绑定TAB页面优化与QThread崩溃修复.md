## 任务目标

优化设置窗体中的"远程绑定"TAB页面，解决以下问题：
1. 已绑定用户 testuser2 但页面显示"未绑定设备"
2. "绑定设备"应改为"绑定用户"（绑定的是账号，与PWA端设备无关）
3. 绑定后应显示用户账号和Token，按钮显示"重新绑定"并需确认
4. "解绑设备"改为"解绑用户"，未绑定时禁用

## 问题诊断与分析

### 问题 1: 绑定状态显示不正确
**现象**: 已绑定用户 testuser2，但页面显示"未绑定设备"
**原因**: 
- 服务器端 `DeviceInfoResponse` 未返回 `username` 字段
- 客户端 `DeviceInfo` dataclass 未包含 `username` 字段
- 状态更新逻辑依赖设备名而非用户名

### 问题 2: QThread 线程崩溃 ⚠️ 严重
**现象**: 打开设置窗体后程序崩溃，日志显示 `QThread: Destroyed while thread 'DeviceStatusLoader' is still running`
**原因**: 
1. **线程生命周期管理不当**：线程在 `__init__` 中通过 `QTimer.singleShot(0, ...)` 启动，此时对话框可能还未完全显示
2. **局部类定义问题**：`DeviceStatusLoader` 类在方法内部定义，可能导致引用问题
3. **parent 设置问题**：线程未正确设置 parent，Qt 对象树无法管理其生命周期
4. **信号连接时机**：对话框关闭时信号可能仍在触发回调

### 问题 3: 文案不准确
**现象**: "绑定设备"、"解绑设备"文案不准确
**原因**: 实际绑定的是用户账号，与用户使用的PWA端设备无关

#### 3.2 使用 showEvent 启动后台任务
```python
def showEvent(self, event) -> None:
    """对话框显示后启动后台任务。"""
    super().showEvent(event)
    
    # 防止重复调用
    if not hasattr(self, '_background_tasks_started'):
        self._background_tasks_started = True
        # 延迟 100ms 启动后台任务，确保对话框完全渲染
        QTimer.singleShot(100, self._initialize_background_tasks)
```

#### 3.3 正确的 closeEvent 处理
```python
def closeEvent(self, event) -> None:
    """对话框关闭时清理后台线程。"""
    self._is_closing = True
    
    if hasattr(self, '_device_loader') and self._device_loader is not None:
        loader = self._device_loader
        if loader.isRunning():
            loader.cancel()
            if not loader.wait(2000):
                logger.warning("设备状态加载线程未能在超时内结束，强制终止")
                loader.terminate()
                loader.wait(500)
    
    super().closeEvent(event)
```

## 技术实现细节

### QThread 生命周期管理的关键点

1. **线程类必须在模块级别定义**
   - 避免局部类定义被提前销毁
   - 确保 Qt 元对象系统正确注册

2. **设置正确的 parent**
   ```python
   # ✅ 正确：设置 parent，Qt 对象树管理生命周期
   self._device_loader = DeviceStatusLoader(server_url, access_token, parent=self)
   
   # ❌ 错误：无 parent，线程对象可能被提前销毁
   self._device_loader = DeviceStatusLoader(server_url, access_token)
   ```

3. **在 showEvent 中启动后台任务**
   - 确保对话框完全显示后再启动
   - 使用 `QTimer.singleShot(100, ...)` 延迟启动

4. **使用 QueuedConnection**
   - 确保信号在主线程中处理
   - 避免跨线程 UI 操作

5. **添加取消机制**
   ```python
   def run(self):
       if self._is_cancelled:
           return
       # ... 执行任务
       if not self._is_cancelled:
           self.finished.emit(result)
   ```

6. **closeEvent 中正确清理**
   - 设置关闭标志
   - 取消线程
   - 等待线程结束
   - 必要时强制终止

## 经验教训

### 1. QThread 生命周期管理 ⚠️ 重要

**问题**：在 `__init__` 中使用 `QTimer.singleShot(0, ...)` 启动线程，对话框关闭时线程仍在运行导致崩溃。

**教训**：
- ❌ 不要在 `__init__` 中启动后台线程
- ✅ 在 `showEvent` 中延迟启动后台任务
- ✅ 线程类必须在模块级别定义
- ✅ 设置正确的 parent 让 Qt 管理生命周期
- ✅ 使用 `QueuedConnection` 确保跨线程信号安全

### 2. 线程清理策略

**问题**：对话框关闭时线程可能仍在运行。

**教训**：
- 设置 `_is_closing` 标志防止回调执行
- 先 `cancel()` 再 `wait()`
- 超时后使用 `terminate()` 强制终止
- 回调中检查 `_is_closing` 状态

### 3. showEvent vs __init__

**问题**：`QTimer.singleShot(0, ...)` 在 `__init__` 返回前就执行。

**教训**：
- `__init__` 中不应启动任何后台任务
- `showEvent` 在对话框完全显示后才调用
- 使用 `_background_tasks_started` 标志防止重复调用

### 4. 信号连接方式

**问题**：直接连接可能在对话框销毁后触发回调。

**教训**：
- 使用 `Qt.ConnectionType.QueuedConnection`
- 回调中检查对话框状态
- closeEvent 中断开信号连接
