# 服务器端日志系统完善方案

## 现状分析

### FastAPI 后端（`winclaw_server/remote_server/`）
**已有基础**：
- ✅ `main.py:28-31` 有基础 logging 配置
- ✅ 各模块使用 `logging.getLogger(__name__)` 记录日志
- ✅ `logs/` 目录已存在

**问题**：
- ❌ 只有控制台输出，没有文件日志
- ❌ 没有日志轮转机制
- ❌ 没有统一的日志配置模块
- ❌ 日志分散，难以集中管理

### PWA 前端（`winclaw_server/pwa/`）
**现状**：
- ❌ 完全没有日志系统
- ❌ 生产环境问题难以追踪
- ❌ 用户操作无记录

## 实施方案

### 阶段一：FastAPI 后端日志系统

#### 1.1 创建统一日志配置模块
**文件**：`winclaw_server/remote_server/logging_config.py`

**核心功能**：
```python
def setup_logging(
    log_dir: str = "logs",
    level: str = "INFO",
    format_type: str = "detailed",
    rotation: str = "daily",
    backup_count: int = 7,
    enable_console: bool = True,
    enable_file: bool = True,
    separate_error: bool = True
) -> None
```

**日志文件结构**：
```
winclaw_server/logs/
├── remote_server.log        # 主日志（最新）
├── remote_server_2026-02-26.log  # 按日期分割
├── error_2026-02-26.log     # 错误日志单独记录
└── archive/                 # 归档目录
```

**日志格式**：
```python
# 详细格式（适合文件）
"%(asctime)s | %(name)-30s | %(levelname)-8s | %(message)s"

# 简单格式（适合控制台）  
"[%(levelname)s] %(name)s: %(message)s"
```

#### 1.2 修改 `main.py` 集成日志
**修改点**：
```python
# 替换原有的 basicConfig
from .logging_config import setup_logging

@asynccontextmanager
async def lifespan(app: FastAPI):
    # 应用启动前设置日志
    setup_logging(
        log_dir="logs",
        level="INFO",
        rotation="daily",
        backup_count=7
    )
    
    logger = logging.getLogger(__name__)
    logger.info("=== WinClaw 远程服务启动 ===")
    # ... 其他初始化代码
```

#### 1.3 添加日志中间件
**文件**：`winclaw_server/remote_server/middleware/logging_middleware.py`

**功能**：
- 记录每个 HTTP 请求
- 记录请求耗时
- 记录请求/响应体（可选）
- 记录用户身份

```python
@app.middleware("http")
async def log_requests(request: Request, call_next):
    start_time = time.time()
    
    # 记录请求
    logger.info(f"{request.method} {request.url.path} - 开始处理")
    
    # 处理请求
    response = await call_next(request)
    
    # 记录耗时
    process_time = time.time() - start_time
    logger.info(f"{request.method} {request.url.path} - 完成 (耗时 {process_time:.3f}s)")
    
    return response
```

### 阶段二：PWA 前端日志系统

#### 2.1 创建前端日志工具类
**文件**：`winclaw_server/pwa/src/utils/logger.ts`

**功能**：
```typescript
class Logger {
  // 日志级别
  static DEBUG = 0;
  static INFO = 1;
  static WARN = 2;
  static ERROR = 3;
  
  // 日志方法
  static debug(message: string, ...args: any[])
  static info(message: string, ...args: any[])
  static warn(message: string, ...args: any[])
  static error(message: string, ...args: any[])
  
  // 发送到后端（可选）
  static async sendToServer(level: string, message: string, context?: object)
}
```

**日志内容**：
- 用户操作（点击、导航等）
- API 请求/响应
- 错误信息
- 性能数据

#### 2.2 集成到关键组件
**需要添加日志的地方**：

1. **API 调用层** (`src/api/*.ts`)
```typescript
// 记录 API 请求
logger.info(`[API] ${method} ${url}`, { params, body });

// 记录错误
logger.error(`[API] 请求失败：${url}`, error);
```

2. **WebSocket 连接** (`src/websocket/*.ts`)
```typescript
logger.info('[WS] WebSocket 连接建立');
logger.warn('[WS] WebSocket 重连中...');
logger.error('[WS] WebSocket 连接断开', error);
```

3. **用户交互** (`src/components/*.vue`)
```typescript
// 记录关键用户操作
logger.info('[UI] 用户发送消息', { sessionId, messageLength });
logger.info('[UI] 切换会话', { oldSession, newSession });
```

4. **错误边界** (`src/App.vue` 等）
```typescript
// 全局错误捕获
onErrorCaptured((error) => {
  logger.error('[App] 未捕获错误', error);
});
```

### 阶段三：前后端日志联动

#### 3.1 PWA 日志上报接口
**文件**：`winclaw_server/remote_server/api/pwa_logs.py`

**接口**：
```python
@router.post("/logs/pwa")
async def receive_pwa_logs(logs: List[PWALogEntry]):
    """接收 PWA 端上报的日志"""
    for log in logs:
        logger.info(f"[PWA] {log.level}: {log.message}", extra={
            "user_id": log.user_id,
            "session_id": log.session_id,
            "timestamp": log.timestamp,
            "context": log.context
        })
```

**数据结构**：
```python
class PWALogEntry(BaseModel):
    level: str  # debug, info, warn, error
    message: str
    user_id: Optional[str]
    session_id: str
    timestamp: datetime
    context: Optional[dict]
    url: str
    user_agent: str
```

#### 3.2 前端配置上报策略
**文件**：`winclaw_server/pwa/src/utils/log_sender.ts`

**策略**：
- **实时上报**：ERROR 级别立即上报
- **批量上报**：DEBUG/INFO/WARN 累积到一定数量或时间后上报
- **离线缓存**：网络异常时暂存 localStorage，恢复后上报
- **隐私过滤**：不上报敏感信息（密码、token 等）

### 阶段五：配置文件


## 注意事项

⚠️ **性能考虑**：
- 生产环境日志级别建议 INFO 或以上
- WebSocket 消息日志默认关闭（量太大）
- PWA 日志上报采用批量策略，避免频繁请求

⚠️ **隐私保护**：
- 不上报敏感信息（密码、token、个人隐私）
- 日志中用户 ID 做脱敏处理（显示前 8 位）
- 考虑 GDPR 合规要求

⚠️ **磁盘空间**：
- 默认保留 7 天日志
- 定期清理旧日志
- 监控日志目录大小
